# Unidad 3.1
Basándonos en nuestro proyecto, pensamos en algunas soluciones para los 4 valores principales del Manifiesto de desarrollo ágil de software. A continuación se mostrará las soluciones para cada problema que se tenga con cada valor mencionado.
## Individuos e interacciones por encima de procesos y herrramientas
Consideramos que en nuestra aplicación el uso de las herramientas es fundamental, ciertamente se necesita de personas que sepan manejar correctamente todas las herramientas que necesitamos para el desarrollo de nuestro software pero consideramos que en algunos momentos puede ser más tardado el hecho de priorizar a los individuos, ciertamente una persona creativa y con la capacidad de innovar puede ayudar mucho al desarrollo de nuestro proyecto, pero no es algo que consideramos que tengamos que priorizar mucho más que alguna otra herramienta que nos facilitaría el trabajo. 

Como una solución a este problema pensamos en tener un ligero tiempo de preparación con las personas que trabaajen en el proyecto para que conozcan correctamente las herrramientas que utilizarán en el proyecto para que luego no haya que priorizar tanto al individuo y todos puedan centrarse mas en los procesos y las herramientas
## Software funcionando por encima de documentación exhaustiva
Para este valor, consideramos que genera un problema en cuanto al software funcional, ya que consideramos que el hecho de priorizar algún prototipo no nos será de mucha utilidad y preferimos tener una buena documentación que generar un prototipo tan temprano. Para este problema en nuestro equipo de desarrollo hemos optado por en vez de mostrar algún prototipo tengamos un mayor enfoque a la documentación, refinándola y siendo precisos con esta, para así conseguir que no sea tan exhaustiva y que nos permita ser de alguna manera ágiles
## Colaboración con el cliente por encima de negociación contractual
En nuestro proyecto es dificil la comunicación continua con el cliente, ocacionando que sea dificil tener una colaboración seguida con este, pensamos en diferentes soluciones para este problema.

Nuestra solución para este tipo de casos es prepararle al cliente una serie de métodos de seguimiento de software, así como software específico para que este sea capaz de tener un seguimiento, pensamos en el uso de un tablero Kanban para que el cliente pueda tener una visión general de lo que hemos estado haciendo y de lo que tenemos pensado hacer de una forma muy resumida para poder seguir captando su interés.
## Respuesta ante el cambio por encima de seguir un plan
Para el cuarto valor pensamos que nuestro equipo de desarrollo tiene un pésimo nivel de organización, algo que dificulta mucho el cambio, es verdad que hay que estar preparado para tener una respuesta ante diversos cambios en el proyecto, pero consideramos que es complicado tener una preparación con la falta de organización que existe. 

Con esto en mente pensamos en dos soluciones que podemos usar para facilitar las respuestas ante el cambio. Pensamos en tener un registro seguido de todos los cambios hechos en el proyecto, también pensamos en el uso de software para la comunicación efectiva entre nosotros.
